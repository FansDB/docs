    Performers

    Warning

    Repeat offenders ignoring correction requests will lose EDIT privileges.

    Mandatory details¶

    To create or submit a new performer, include the following minimum required information:

    • Unique primary name(1)
    • Gender(2)
    • At least 1 live or defunct1 link from approved network2(3)
    • Usernames as aliases from submitted approved network2 links(4)
    1. Using FirstName LastName is preferred, but unique username can also be used.
    2. In cases where a performer identifies differently in their daily life, use the gender they use for adult content production.
    3. We rely on these networks for performer age verification.
    4. Helps avoid duplicate submissions and improves discoverability.

    Primary name & aliases¶

    1. Use FirstName LastName as the primary name, or fall back to the unique username from the approved network.
    2. Include usernames as aliases of added performer accounts from approved networks.
    3. Names from professional adult credits can be added as aliases.
    4. Social media accounts can be used to verify other names associated with the adult work. Must be properly documented in the comments.
    5. Do not add social media or other non-network usernames as aliases.
    6. Do not add legal names unless they were used as an adult credit.
    7. Properly source each name and alias. Either by submitted links or in the comments.(1)
    8. Name and alias fields are case-insensitive; do not add different capitalization variations.
    1. If you can't source the information, exclude it from your submission.

    Links¶

    1. Only links that have a matching site can be added.
    2. Defunct links can be added and should not be removed.(1)
    3. Defunct links from approved networks require additional proof.1
    4. Links that redirect to a different account, should include both links. Except in rare cases where they redirect to an unrelated performer.
    5. Avoid adding shortened links. If shortened URL redirects to a full link, use that instead.
    1. Defunct links are kept for prosperity and ability to lookup information from archives.

    Tip

    When adding links, make sure to click Add when adding links manually.

    Suggest a site

    If there is a site you would like to be added, you can suggest them in this document and we will review it.

    Images¶

    1. The main purpose of images is to identify the performer.
    2. Images must have only the performer and no other people.
    3. Try using a variety of different image shot types, ranging from portrait shots to full body shots.(1)
    4. Do not add images where performer face is not visible, except in cases where no pictures with the performer's face exist.
    5. Do not add images that have artificial filters applied to them.
    6. Do not add animated images.
    7. Do not add image compilations/collages.
    8. Do not add images that have 3rd-party watermarks.
    1. The main purpose of these images is to identify the performer. It shouldn't be used as a way to host your favorites.

    Couple type accounts¶

    Info

    We only allow individual profiles for each performer. But couple accounts can be used to verify multiple performers in limited circumstances.

    Due to the imprecise nature of these accounts, it should only be done by experienced editors.

    The submission must provide all relevant details to be able to qualify the account or it will be rejected.

    Account can qualify based on following criteria:

    1. Performers are featured together in the majority of the content.
    2. Performers are featured solo in less than half the content.
    3. One of the performers is featured with others in less than half the content.
    4. Performers are in a legal marital relationship and performed at least once together.

    If they qualify:

    1. Add account username as an alias to all qualifiying profiles.
    2. Add network link of the account to all qualifiying profiles.
    3. Add account username to the disambiguation to all qualifiying profiles in the following format: username, relationship-type where relationship-type can be sister, brother, mother, father, wife, husband, partner.

    1. Defunct links can be added, but they require additional proof. Such proof can be a screenshot of watermarked content, social media link to a post or profile where the performer themselves promoting their account or a working archived copies of their account page (e.g. archive.org). ↩↩

    2. See Networks for a list of approved networks. ↩↩
